# aruaru volunteer / GT730 development context

aruaruは、ハイエンドGPUや高額な開発予算がない状況でも、Rust + Poem + GraphQL / WunderGraph Cosmo系のアプリ開発、README自動生成、自動BUGチェック、AI連動開発を進めるためのプロジェクトです。

## 開発背景

このプロジェクトは、無職・無報酬のボランティア状態でも進められるように、次の制約を前提に設計されています。

- GT730級の低性能GPU環境でも作業できる。
- 生成AI代、ドメイン更新料、VPSレンタルサーバー代などを自腹で支払う厳しい状態でも継続できる。
- 高額なLLM APIを常用せず、安いチェックを先に行う。
- paid APIは、ローカルチェックで解けない難問だけに使う。
- 手動スクリプトでも同じ品質ゲートを再現できる。

## 技術方針

1. 低コスト優先
   - cargo fmt / check / test / clippyを先に実行する。
   - LLMに投げる前にログを圧縮する。

2. ローカル優先
   - ローカルLLM、aruaru-llm、iLumi.llm、GGUF/Ollama/LM Studioを優先する。
   - 既にダウンロード済みのモデルは再ダウンロードしない。

3. API最小利用
   - Opus / ChatGPT / Claude / GeminiなどはAPI接続として扱う。
   - APIに送る前に秘密情報を除外する。

4. 品質ゲート重視
   - 自動修正よりも差分確認と再検証を優先する。
   - ユーザー承認なしに修正を適用しない。

## READMEでの表現

READMEでは、この背景を「苦しい事情の暴露」だけではなく、低コスト・個人開発・ボランティア開発でも品質ゲートを維持するための設計理由として記載します。
