# 開発方針

## 基本方針

`aruaru` は、低コストでも段階的に成長できる、AIネイティブな開発・運用基盤を目指します。

## 技術方針

- Rust + Poem を重要候補とする
- JavaScript ではなく TypeScript を使用する
- REST API は基本的に使わず、WunderGraph Cosmo を代替候補にする
- PostgreSQL 連携を重視する
- 将来的に aruaru-db 独自DBも構想する
- 分散DB、VersionlessAPI、移植変換、分散自動バックアップ、Git型DB履歴を設計に含める

## 品質方針

些細なミス削減、BUG削減、品質ゲート強化を重視します。

AIの初回出力をそのまま採用せず、以下を確認します。

- 問題点
- 原因
- 修正方針
- 確認方法
- コンパイル確認
- テスト確認
- 型チェック
- 回帰テスト
- セキュリティ確認

## 低スペックPC対応

現時点では、開発者が生活困難で十分な開発資金がなく、GT 730 搭載PCのまま開発を進める前提があります。

そのため、以下を重視します。

- 軽量構成
- 小さく始める設計
- 古いPCでも編集・確認しやすい構成
- 重いAI処理はクラウドAIや外部AIも併用
- ローカルLLMは環境に応じて自動選択
- 後で高性能PCへ移行しても無駄にならない設計

## 確認方法

- GT 730 PCでも設計書編集・Git操作が可能か
- 重い処理を必須にしていないか
- ローカルLLMを必須にしていないか
- クラウドAIとローカルAIを切り替えられるか
